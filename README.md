# Primitive DB — надежная файловая СУБД на Python

**Primitive DB** — это учебный проект консольной базы данных, реализующий хранение данных в формате JSON. В отличие от простых скриптов, данный проект построен на принципах модульности, строгой типизации и безопасности данных.

## ✨ Ключевые особенности

### 🏗 Архитектура и качество кода

* **Модульная структура:** Логика четко разделена между ядром, движком, парсером и слоем хранения.
* **Современный Python:** Активное использование `match/case` (Python 3.10+) для обработки команд и условий.
* **Clean Code:** Полное соответствие PEP8, семантическое именование переменных и подробные docstrings.

### 🛠 Управление данными (DDL & CRUD)

* **Управление таблицами:** Создание схем с типами (`create_table`), удаление (`drop_table`) и просмотр (`list_tables`).
* **Полный CRUD:** * `insert`: автоматическая генерация уникальных **ID**.
* `select`: мощная фильтрация через блок `where`.
* `update`: массовое или точечное обновление через `set`.
* `delete`: безопасное удаление записей.



### 🚀 Продвинутые механики

* **Атомарная запись:** Использование временных файлов (`.tmp`) при сохранении JSON гарантирует, что база данных не повредится при сбое питания.
* **Интеллектуальное кэширование:** Реализовано через **замыкания**. Кэш автоматически сбрасывается (инвалидируется) только при изменении данных в конкретной таблице.
* **Система декораторов:**
* `@catch_database_errors`: централизованная обработка исключений.
* `@audit_log_command`: логирование всех действий в `logs/commands.log`.
* `@measure_execution_time`: точный замер производительности команд.
* `@require_confirmation`: защита от случайного удаления данных.

## 📂 Структура проекта

```text
src/primitive_db/
 ├── main.py         # REPL (точка входа) и цикл обработки ввода
 ├── engine.py       # Диспетчер команд и логика вывода таблиц
 ├── core.py         # Ядро (JsonDatabaseCore): логика CRUD и кэширования
 ├── parser.py       # Парсер команд (Regex + логика WHERE/SET)
 ├── storage.py      # Слой хранения: атомарная работа с файловой системой
 ├── decorators.py   # Набор функциональных оберток (Логи, Время, Ошибки)
 ├── utils.py        # Утилиты: типизация, валидация, форматирование PrettyTable
 ├── errors.py       # Доменная иерархия исключений (DBError)
 └── constants.py    # Глобальные настройки, пути и системные сообщения

```

## ⌨️ Синтаксис команд

| Команда | Синтаксис | Пример |
| --- | --- | --- |
| **Создание** | `create_table <table> <field:type> ...` | `create_table user name:str age:int` |
| **Добавление** | `insert <table> <field=value> ...` | `insert user name="Ivan" age=25` |
| **Выборка** | `select <table> [where <cond>]` | `select user where age>20` |
| **Обновление** | `update <table> set <f=v> [where <cond>]` | `update user set age=26 where id=1` |
| **Удаление** | `delete <table> [where <cond>]` | `delete user where id=1` |

**Поддерживаемые типы:** `int`, `float`, `str`, `bool` (true/false, yes/no, 1/0).
